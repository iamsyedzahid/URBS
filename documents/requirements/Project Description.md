Project Description
In your course project, you are required to develop a suitable MVP or a prototype of a software system that should be fit for actual users of such a system. You need to gather requirements from actual users as well as research similar existing projects in order to propose a system that can solve your customers’ requirements. You need to follow all stages of the SDLC as discussed during the course and provide different artifacts as milestones. 


The project will be graded on the following criteria:
1.	Quality of requirement engineering, including tools and techniques used for RE.
2.	The suitability and effectiveness of the proposed process model used for the project.
3.	Distribution and allocation of work among the group members.
4.	Choosing the part/ features of the project to focus on during development. It should be the ones that are the most important to achieve the objectives of the product.
5.	Quality of development practices.
6.	Verification and Validation of your project.
7.	A workable application with all the implemented functionalities that you have listed in Abstract and in SRS
8.	Input validations are mandatory for every information management application
9.	All group members need to be there for the project evaluation
10.	Project assessment will be done individually
